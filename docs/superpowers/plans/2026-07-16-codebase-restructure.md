# Codebase Restructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the app into a `src/` layout, add ESLint, archive design
reference material, and reserve a folder for the upcoming AI backend -
following `docs/superpowers/specs/2026-07-16-codebase-restructure-design.md`.

**Architecture:** Pure structural/config change. `git mv` for every relocated
file so history is preserved as renames, not delete+recreate. No component
logic, content data, or chatbot text changes anywhere in this plan.

**Tech Stack:** Next.js 16 (App Router), TypeScript 5.8, Tailwind CSS 4,
ESLint (`eslint-config-next`).

## Global Constraints

- This repo IS a git repository now (`git init` already run, several commits
  exist) - every move must use `git mv`, and every task ends with a commit.
- `content/*.json` and `context/*.md` stay at repo root - do not move them.
- `metadata.json` and `.impeccable/` stay untouched at repo root.
- No behavior change: every route (`/`, `/portfolio`, `/projects/[slug]`,
  `/writing/[slug]`) must render identically before and after.
- Verification for every task is `npx tsc --noEmit` (must show
  `TypeScript: No errors found`) - there is no test framework in this repo.

### Task 1: Move `app/` and `components/` into `src/`, update tsconfig aliases

**Files:**
- Move: `app/` → `src/app/` (all files, via `git mv`)
- Move: `components/` → `src/components/` (all files, via `git mv`)
- Modify: `tsconfig.json`

**Interfaces:**
- Consumes: nothing (first structural task).
- Produces: `src/app/`, `src/components/` as the new locations every later
  task and the future backend route (`src/app/api/chat/route.ts`) build
  under. The `@/*` alias now resolves into `src/`, with `@/content/*` and
  `@/context/*` as separate aliases pointing at the untouched root
  directories.

- [ ] **Step 1: Confirm current git status is clean**

Run: `git status`
Expected: `nothing to commit, working tree clean` (if not, stop and report -
do not move files on top of uncommitted work).

- [ ] **Step 2: Create `src/` and move `app/` and `components/` into it**

```bash
mkdir -p "src"
git mv app src/app
git mv components src/components
```

- [ ] **Step 3: Update `tsconfig.json` paths**

In `tsconfig.json`, replace:

```json
    "paths": {
      "@/*": [
        "./*"
      ]
    }
```

with:

```json
    "paths": {
      "@/*": [
        "./src/*"
      ],
      "@/content/*": [
        "./content/*"
      ],
      "@/context/*": [
        "./context/*"
      ]
    }
```

- [ ] **Step 4: Verify TypeScript resolves the moved files**

Run: `npx tsc --noEmit`
Expected: `TypeScript: No errors found`. If it fails on `@/content/...` or
`@/context/...` imports, check step 3 was applied exactly - those two
aliases must exist alongside `@/*`.

- [ ] **Step 5: Verify the dev server serves all routes**

Run: `npm run dev`, then in a browser check:
- `http://localhost:3000/` (chat landing page)
- `http://localhost:3000/portfolio` (full site, all sections)
- `http://localhost:3000/projects/<any-existing-slug>` (pick any slug from
  `content/projects.json`)
- `http://localhost:3000/writing/<any-existing-slug>` (pick any slug from
  `content/writing.json`)

Expected: all four routes render with no 404s, no red error overlay, no
missing-module console errors. Stop the dev server after checking.

- [ ] **Step 6: Commit**

```bash
git add -A
git commit -m "refactor: move app and components into src/ layout"
```

### Task 2: Add ESLint (`eslint-config-next`)

**Files:**
- Create: `eslint.config.mjs`
- Modify: `package.json`
- Modify: `package-lock.json` (regenerated by `npm install`)

**Interfaces:**
- Consumes: `src/app/`, `src/components/` from Task 1 (lint target paths).
- Produces: `npm run lint:eslint` script, usable by any future CI step or
  developer workflow. Does not touch the existing `lint` script (`tsc
  --noEmit`), which stays as the type-check command.

- [ ] **Step 1: Install `eslint-config-next` and `eslint`**

Run: `npm install --save-dev eslint eslint-config-next`
Expected: install succeeds, `package.json` devDependencies gains both
packages.

- [ ] **Step 2: Create `eslint.config.mjs`**

```javascript
import { FlatCompat } from "@eslint/eslintrc";

const compat = new FlatCompat({
  baseDirectory: import.meta.dirname,
});

const eslintConfig = [
  ...compat.extends("next/core-web-vitals", "next/typescript"),
];

export default eslintConfig;
```

- [ ] **Step 3: Add the `lint:eslint` script**

In `package.json`, the `"scripts"` block currently reads:

```json
  "scripts": {
    "dev": "next dev --port 3000",
    "build": "next build",
    "start": "next start",
    "clean": "rm -rf .next",
    "lint": "tsc --noEmit"
  },
```

Replace with:

```json
  "scripts": {
    "dev": "next dev --port 3000",
    "build": "next build",
    "start": "next start",
    "clean": "rm -rf .next",
    "lint": "tsc --noEmit",
    "lint:eslint": "eslint ."
  },
```

- [ ] **Step 4: Run ESLint**

Run: `npm run lint:eslint`
Expected: no errors. Warnings on pre-existing code are acceptable to leave
as-is (fixing unrelated lint warnings is out of scope for this plan) - but
if it errors, read the message and fix only what's necessary to get a clean
run (e.g. a missing `eslint-plugin-*` peer dependency `npm` reports).

- [ ] **Step 5: Verify the type-check still passes**

Run: `npx tsc --noEmit`
Expected: `TypeScript: No errors found`

- [ ] **Step 6: Commit**

```bash
git add eslint.config.mjs package.json package-lock.json
git commit -m "chore: add eslint-config-next for linting"
```
